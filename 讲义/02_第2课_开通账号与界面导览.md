# 第 2 课　开通账号与界面导览

|**🎯 本课目标**<br>1\. 搞清两种使用 Dify 的方式，并选对适合自己的；<br>2\. 顺利完成注册 / 登录，进入工作台；<br>3\. 认全主界面的每一个区域，做到“想找功能不迷路”；<br>4\. 提前认识工作室里能创建的 5 种应用，心里有底。|

## 2\.1　两种使用方式，先选一个

用 Dify 有两条路。新手强烈建议先走第一条（云端版），等熟练了再考虑第二条：

|**方式**|**适合谁**|**优点**|**缺点**|
|---|---|---|---|
|云端版（官网注册直接用）|所有新手、想快速上手的人|打开网页就能用，零安装、自动更新|需要联网，免费额度有限|
|社区版 / 自部署（装在自己电脑或服务器）|懂点技术、数据要私密、要深度定制|数据完全在自己手里，可定制、可商用|要装环境（Docker），门槛较高|

**怎么选？**一句话——只要不是“资料绝对不能上云”的硬性要求，就先用云端版。它省掉了一切安装和运维的麻烦，让你把精力全花在“学会做应用”上。本课程后续操作，默认都以云端版讲解。

|**💡 小贴士**<br>Dify 是开源项目，名字取自 “Do It For You（替你做）”，在 GitHub 上是最受欢迎的 AI 应用开发平台之一。开源意味着你随时可以把它免费下载到自己服务器上私有部署，这一点对企业很有吸引力。|
|---|

## 2\.2　注册并登录（云端版）

整个过程和注册一个普通网站差不多，几分钟就能搞定：

1. 打开浏览器，访问 Dify 官网（dify\.ai），点击右上角的“开始使用 / Get Started”。

2. 选择注册方式：可以用邮箱注册，也可以直接用 GitHub 或 Google 账号一键登录（更快）。

3. 若用邮箱注册，去你的邮箱点开确认链接完成验证；个别邮件可能在“垃圾邮件”里。

4. 首次登录后，系统会引导你创建或进入一个“工作区（Workspace）”——可以理解成你的专属工作空间。

5. 进入后看到的主页面，就是你以后的主战场。如果界面是英文，可在“设置 → 语言”里切换成中文。

|**⚠️ 避坑提醒**<br>如果收不到验证邮件：先翻一遍垃圾邮件箱；部分网络环境访问官网较慢，多刷新几次或换个时段再试即可。|
|---|

**关于“工作区”：**它是用来归置你所有应用、知识库的空间，也方便日后拉同事进来协作。个人学习阶段，用默认的工作区就行，不必纠结。

顶部四大菜单，逐个认识一下：

|**菜单**|**点进去能做什么**|**一句话记住**|
|---|---|---|
|探索 / Explore|浏览官方做好的应用模板，可一键复制来改|找灵感、抄作业的地方|

**右上角的头像 / 设置** 是另一个关键入口：模型供应商、API 密钥、成员管理、语言等都在这里配置。第 3 课接模型就会用到它。

**新手动线建议：**先去“设置 → 模型供应商”把大模型接好（第 3 课），再回“工作室”点“创建应用”动手做（第 4 课）。

## 2\.4　提前认识：工作室能建的 5 种应用

在“工作室”里点“创建应用”，会让你选应用类型。现在先混个脸熟，知道各自是干嘛的即可，后面课程会逐个实操：

|**应用类型**|**一句话定位**|**典型场景**|
|---|---|---|
|聊天助手|最常见的多轮对话机器人，能记上下文、可挂知识库|客服、咨询助手、专属问答|
|文本生成应用|填好表单、一次性出结果的“无对话”工具|周报、文案、翻译、摘要|
|Agent（智能体）|能自己拆解任务、调用工具、反复尝试|需要查资料、算数、调外部接口的任务|
|Chatflow（对话流）|带记忆的、可编排复杂流程的多轮对话|多步骤的复杂客服 / 表单收集|
|工作流（Workflow）|面向自动化、批处理的单向流程，常配合接口使用|批量处理、数据加工、后台自动化|

|**💡 小贴士**<br>别被五种类型吓到。新手只要先掌握前两种——“聊天助手”和“文本生成”——就能做出非常多实用的东西了。后面三种是进阶玩法，循序渐进即可。|
|---|

## 2\.5　善用“探索”：站在别人的肩膀上

“探索 / Explore”里有大量官方和社区做好的模板应用。对新手来说，这是最好的学习捷径：

- 先打开几个模板，体验一下成品长什么样、能干什么；

- 看中了哪个，就把它“添加/复制”到自己的工作室，再打开看它内部是怎么配置的；

- 照着改成自己的版本——这比从零开始快得多，也学得快。

|**⚠️ 避坑提醒**<br>光看模板会有“一看就会、一做就废”的错觉。每节课的“课堂练习”一定要亲手做一遍，手感是练出来的，不是看出来的。|
|---|

## 2\.6　社区版自部署是怎么回事

如果你的数据很敏感、不想放云端，可以把 Dify 装在自己的电脑或服务器上。最常见的方式是用 Docker（一种“软件集装箱”工具，能把 Dify 运行所需的一堆东西打包好一键启动）。大致步骤是：

1. 在自己的电脑 / 服务器上安装 Docker（桌面版或服务器版）。

2. 从官方仓库下载 Dify 的代码包（含一份 docker 配置）。

下载地址：https://github\.com/langgenius/dify/

3. 进入对应目录，在命令行运行一行启动命令，等它把各项服务跑起来。

注意需要的系统环境

- CPU \>= 2 Core

- RAM \>= 4 GiB

windows系统:

```Plain Text
git clone https://github.com/langgenius/dify/
cd dify
cd docker
copy .env.example .env  
docker compose up -d
```

4. 用浏览器访问本机地址（如 http://localhost），首次会让你初始化管理员账号。

5. 注意，默认的是80端口，如果docker 启动不起来，判断是否端口被占用了

|**⚠️ 避坑提醒**<br>自部署涉及命令行操作，属于进阶内容，对电脑配置也有一定要求。第一次学 Dify，强烈建议先用云端版，把功能玩熟，别在“装环境”上卡住而失去信心。|
|---|

|**❓ 常见疑问**<br>问：云端版安全吗？我的数据会被看到吗？<br>答：云端版里你的数据存在 Dify 的服务器上，正规平台有相应的安全措施。但如果你处理的是高度敏感的内部资料，对合规要求严格，建议直接用社区版自部署，数据完全留在自己手里。|
|---|

|**✏️ 课堂练习**<br>1\. 完成账号注册并成功登录，进入工作室；如果是英文界面，把它切换成中文。<br>2\. 把顶部四个主菜单依次点一遍（先看不操作），再点开右上角“设置”看看里面有什么，建立空间感。<br>3\. 去“探索 / Explore”随便打开一个官方模板应用，体验一下别人做的成品，并把它复制到自己的工作室留着研究。|

**本课小结：**新手用云端版最省心；顶部四大菜单记成“找灵感 / 做东西 / 喂资料 / 装手脚”，配置入口在右上角设置；工作室能建 5 种应用，先掌握“聊天助手”和“文本生成”即可；多用“探索”模板抄作业、但一定要亲手练。
