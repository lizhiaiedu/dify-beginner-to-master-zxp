## 3\.1　为什么第一步是接模型

还记得第 1 课的比喻吗？大模型是发动机。Dify 这辆车造得再漂亮，**没装发动机也开不动**。Dify 本身并不自带大模型，它是个“编排平台”，必须由你接入外部模型，应用才能真正会“思考”和“说话”。

所以做任何应用之前，第一件事永远是：在 Dify 里接好至少一个大模型。这一步做不好，后面新建的应用会“选不到模型”，什么都跑不起来。

## 3\.2　先搞懂 API 和 API Key

**API**（应用程序接口）可以理解成两个软件之间“点菜的窗口”。Dify 想用某家的大模型，不需要把模型搬到自己家，只要通过对方开放的 API 这个“窗口”去下单：把问题递进去，把回答端出来。

**API Key**（接口密钥）则是你在这个窗口的“会员钥匙 \+ 记账卡”。你向模型厂商申请一把钥匙，Dify 每次点菜都出示它，对方据此确认“是你在用”，并把消费记到你的账上。

## 3\.4　实操（上）：去厂商那里拿钥匙

下面讲通用流程。无论你选哪家，套路都大同小异：

1. 注册并登录一家模型厂商的开放平台（如 DeepSeek 开放平台、阿里云百炼等）。

2. 在后台找到“API Keys / 密钥管理”菜单，点击“创建新密钥”。

3. 给密钥起个好认的名字（如 dify\-test），创建后把那一长串字符立即复制下来。

4. 先把它粘到记事本临时存好——很多平台的完整密钥只在创建时显示这一次。

|**⚠️ 避坑提醒**<br>如果某些平台需要先“实名认证”或“开通服务 / 充值才送额度”，按它的提示走一遍即可，这是厂商侧的要求，不是 Dify 的问题。|
|---|

## 3\.6　三类模型，别搞混

配置页里你会看到模型分好几类，它们各司其职：

|**类型**|**干什么用**|**什么时候需要**|
|---|---|---|
|对话模型（LLM）|理解问题、生成回答|做任何应用都需要，必接|
|向量模型（Embedding）|把文字转成数字向量，方便“找资料”|做知识库（第 7 课）时需要|
|重排序模型（Rerank）|对检索出的资料做二次排序，提升精准度|知识库进阶优化（选配）|

## 3\.7　接不通？对照排查清单

如果状态没变“可用”，或新建应用时选不到模型，按下面几条逐一检查，基本都能解决：

- **Key 复制错了：**首尾多了空格、漏了字符，重新复制粘贴一次。

- **没充值 / 没领额度：**部分厂商要先实名或开通服务才有额度，去后台确认。

- **没装对供应商：**确认“安装”的厂商和你申请 Key 的厂商是同一家。

- **没设默认模型：**回到“系统模型设置”把推理模型选上。

- **网络问题：**海外模型（OpenAI 等）可能需要特定网络环境，新手建议先用国产模型。

## 3\.8　本地部署模型

前面接的都是厂商云端的模型，钥匙拿来即用。但有时你会想把模型跑在自己的机器上——数据不出门、不按 token 计费、断网也能用。这就是“本地部署”。Dify 同样支持把本地模型接进来，常见两种方式：Ollama（适合新手，开箱即用）和 vLLM（适合追求性能、有显卡的进阶玩家）。

下面分别说，但请先记住一个贯穿全程的关键点——

|**⚠️ 避坑提醒**<br>地址别填 localhost Dify 是跑在 Docker 容器里的。容器里的“localhost”指的是容器自己，不是你这台电脑（宿主机）。所以即使模型就在本机跑着，在 Dify 里也不能填 `http://localhost``:端口`，否则一定连不上。正确填法：<br>- Windows / Mac（Docker Desktop）：用 `http://host.docker.internal``:端口`，这是 Docker 专门指向宿主机的地址。<br>- Linux：用宿主机在 docker 网桥上的 IP（常见 `http://172.17.0.1``:端口`），或直接填局域网 IP。<br>- 模型在另一台机器：直接填那台机器的 IP。|
|---|

### vLLM部署模型接入

vLLM 启动后自带“OpenAI 兼容接口”，所以在 Dify 里不用找专门的 vLLM 选项，而是用通用的“OpenAI\-API\-compatible”供应商。

1. 在有显卡的机器上启动 vLLM 服务，注意两个参数：`--host 0.0.0.0`（让容器连得进来）、`--served-model-name`（给模型起个对外名字）。

```Plain Text
docker run --runtime nvidia --gpus all `
    -v D:/modelscope:/root/.cache/modelscope `
    -p 8000:8000 `
    --ipc=host `
    -e VLLM_USE_MODELSCOPE=true `
    vllm/vllm-openai:latest `
    --model Qwen/Qwen2.5-0.5B-Instruct `
    --served-model-name qwen2.5-0.5b `
    --max-model-len 4096 `
    --gpu-memory-utilization 0.5
```

`--runtime nvidia` —— 指定用 NVIDIA 容器运行时,让容器能调用显卡驱动

`--gpus all` —— 把宿主机所有 GPU 都分配给这个容器使用。可以改成 `--gpus '"device=0"'` 只用第 0 块卡

`-v D:/modelscope:/root/.cache/modelscope` —— 目录挂载\(volume\)。冒号左边 `D:/modelscope` 是你宿主机的目录,右边 `/root/.cache/modelscope` 是容器内部的路径。作用是把模型下载缓存放到宿主机的 D 盘,这样容器删了重建,模型不用重新下载

`-p 8000:8000` —— 端口映射\(port\)。左边是宿主机端口,右边是容器内端口。这样你在宿主机访问

`--ipc=host` 是让容器**共享宿主机的 IPC\(进程间通信\)命名空间**,核心作用是让容器能用宿主机的**共享内存\(/dev/shm\)**

`-e VLLM_USE_MODELSCOPE=true` —— 设置容器内的环境变量\(environment\)。这里让 vLLM 从魔搭社区下载模型

`--max-model-len 4096` —— 最大上下文长度\(输入\+输出的 token 上限\)。调小能省显存;调大能处理更长文本但更吃显存

`--gpu-memory-utilization 0.5` —— vLLM 允许占用的显存比例,这里是 50%。vLLM 会预先占住这部分显存做 KV 缓存。显存小就调低\(如 0\.5\),想榨干性能可调到 0\.9

2. 启动验证  curl http://localhost:8000/v1/models

3. 回到 Dify：设置 → 模型供应商 → 找到“OpenAI\-API\-compatible”，点添加。

![Image](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=ODc3MGI0MTBiMTQ3ZDcyY2FhODAwMjhhNjVhZTg2NDNfZmI1NjE4NjQ5ZmUwOWYwN2Y2Y2VlNDIzYzNjN2ZhOWVfSUQ6NzY1MDExMjM2MjU3NzQzMTQ5Ml8xNzgyMTk2NjIyOjE3ODIyODMwMjJfVjM)

4. 填三项： 

    - Model Name：填 `qwen2.5-0.5b`（必须和 `--served-model-name` 一字不差）。

    - API Base / 接口地址：`http://host.docker.internal:8000/v1`（结尾的 `/v1` 别漏）只改端口，其他不变。

    - API Key：vLLM 默认不校验，随便填个非空字符串如 `EMPTY` 即可。

5. 保存，看到“可用”绿点即接通。

### Ollama部署模型接入

Ollama 更省心，下载即用，Dify 里也有专门的“Ollama”供应商。

一定要用本地部署的docker接入模型

1. 在本机安装并启动 Ollama，让它监听所有网卡（否则容器连不进来）：设置环境变量 `OLLAMA_HOST=0.0.0.0:11434`，然后重启 Ollama。

```Python
#mac安装
brew install ollama
# windows 安装
pip install ollama
#设置环境变量
launchctl setenv OLLAMA_HOST "0.0.0.0"
# mac重启ollama
brew services restart ollama
```

2. 先把模型拉下来跑起来，例如：`ollama run qwen2.5`。用 `ollama list` 能看到已装的模型名。

```Python
# 拉取模型 
ollama pull qwen2.5  
# 查看已安装模型 
ollama list  
# 测试运行 
ollama run qwen2.5
```

3. 测试是否成功 

```Bash
curl http://localhost:11434
```

返回 "Ollama is running" 即成功

4. 回到 Dify：设置 → 模型供应商 → 找到“Ollama”，点添加。

5. 填两项： 

    - Model Name：填 `qwen2.5`（和 `ollama list` 里完全一致）。

    - Base URL：`http://host.docker.internal:11434`。

6. 保存，看到“可用”即接通。
