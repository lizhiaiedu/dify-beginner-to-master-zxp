## 4\.2　聊天助手的两种编排方式

创建聊天助手时，Dify 会让你选“编排方式”，新手容易在这里迷糊，先讲清楚：

|**编排方式**|**是什么**|**什么时候用**|
|---|---|---|
|基础编排|就一个提示词框 \+ 一个模型，最简单|本课、绝大多数入门场景|
|工作流编排（Chatflow）|在每轮回复背后跑一条多节点流程|需要查资料、分支判断等复杂逻辑（第 9 课起）|

|**💡 小贴士**<br>记住一句话：先用“基础编排”把东西做出来，等它满足不了需求（比如要挂知识库、要分流）时，再升级到工作流编排。不要一上来就追求复杂。|
|---|

## 4\.3　实操：创建一个“英语学习小助手”

目标：做一个能帮你练英语、随时答疑、语气友好的聊天助手。跟着做：

1. 进入“工作室”，点击“创建空白应用”。

2. 应用类型选“聊天助手 / Chatbot”，编排方式选“基础编排”。

3. 给应用起名，比如“英语学习小助手”，可以选个图标，点“创建”。

4. 进入编排页，最重要的是中间那个大输入框——“提示词 / 系统提示词”。

5. 在提示词框里填入助手的“人设和任务”，例如下面这段：

|**示例·系统提示词**<br>*你是一位耐心、友好的英语老师。当用户用中文或英文向你提问时，你要：① 用最简单的话解释清楚；② 给出 2 个地道的例句；③ 最后用一句话鼓励用户多开口。回答尽量简洁，不超过 150 字。*|
|---|

1. 在右上角的模型选择处，确认选中了第 3 课接好的对话模型。

2. 在右侧“调试与预览”里输入一句话试试，比如“apple 怎么用”，看看它的回答是否符合你设定的人设。

3. 不满意就回到提示词框接着改，改完再测——这就是后面第 5 课要专门练的“打磨”。



### 系统提示词、用户提示词、助手提示词区别

**1\. 系统提示词（System）—— 设定人设和规则，你写一次，全程生效**

这是给 AI 定身份的，相当于"开机说明书"。在对话最开始就生效，并且贯穿整段对话。你在这里写：它是谁、什么风格、要遵守什么规则、不能做什么。**学习小助手的人设就写在这里。** 用户看不到这部分。

**2\. 用户提示词（User）—— 用户每一轮说的话**

就是使用者真正输入的内容，比如"系统提示词 英语是什么"。每轮对话用户说一次，是动态变化的。

**3\. 助手提示词（Assistant）—— AI 的回复**

就是 AI 说的话。它有两个用途：

- 正常对话里，它是模型**生成**的回答；

- 但你也可以**手动预填几条**，当作"示范样例"教模型——告诉它"理想的回答应该长这样"。这叫少样本示例（few\-shot），是控制回答风格、格式最有效的手段之一。

例如：System：你是一个情感专家\.\.\.（开头一次，常驻）   

User：（示范）我男朋友总不回消息 

Assistant：（示范）听起来这种被晾着的感觉挺难受的\.\.\.   ← 你预填，教风格



## 4\.4　场景模拟

这个需求非常实战！我们来搭建一个\*\*“智能出差/外勤行政助手”\*\*。

### 场景模拟：企业智能助手

当员工输入一段话时，助手会自动判断：

1. **出差/外勤查询**：如果是出差，自动查询天气，并根据天气给出打车或着装建议。

2. **日常工作总结**：如果是汇报工作，自动套用周报格式进行整理。

3. **其他闲聊**：礼貌引导员工回到工作话题。

### 工作流搭建全过程

#### 1\. 进入 Dify，创建工作流 \(Workflow\)

命名为：“行政办公百事通”。

#### 2\. 节点详细设置

（1）添加输入变量：`user_request` \(段落\)

（2）问题分类

**作用**：把员工的需求分流。

**设置分类**：

1. **`出差/外勤查询`**：包含出差、外勤、去某个城市、天气查询等。

2. **`日常工作总结`**：包含写日报、周报、工作总结、任务进度等。

**输出：出差/外勤查询、日常工作总结、其他闲聊（会三选一输出）**

（3）分支一

**A\. 天气查询**

- **输入变量**：`City` 引用 **节点 C** 输出的城市名。

- **作用**：从用户的话里提取“城市”。因为天气工具需要一个纯净的城市名。方便更具天气决定出行方式

- **提示词**：`请从 {{arg0}} 中只提取城市名称（如：北京），不要输出任何其他字符。`

- **变量**：`arg0` 引用 `开始.user_request`。

- **选择工具**：在工具栏搜索“高德天气”或“搜索工具/GoogleSearch”。

- 输出格式：

```Python
{
  "text": "{\"text\": \"烟台：晴。温度 23.0°C，体感 17.0°C，湿度 46%，风速 8.0 公里/小时。\"}",
  "files": [],
  "json": [
    {
      "text": "烟台：晴。温度 23.0°C，体感 17.0°C，湿度 46%，风速 8.0 公里/小时。"
    }
  ]
}
```

**B\. 代码转换参数**

- **作用**：根据天气决定建议。

- **分支设置**：

    - **IF**：`天气状况` \(来自节点 D\) **包含** "雨"。

    - **ELSE**：\(默认路径\)。

- **作用**：把天气数据和建议拼接成漂亮的工作简报。

- **输入变量**：`city`, `weather`, `temp`, `is_rain` \(来自条件判断的布尔值\)。

- **代码如下**：

入口函数必须叫 `main`，这是平台规定的，名字不能改。

```Python
import re

def _find_text(obj):
    """递归找出那句天气文本，无论被 list/dict 怎么包"""
    if isinstance(obj, str):
        return obj if "：" in obj and "温度" in obj else None
    if isinstance(obj, dict):
        for v in obj.values():
            r = _find_text(v)
            if r:
                return r
    if isinstance(obj, list):
        for v in obj:
            r = _find_text(v)
            if r:
                return r
    return None

def main(args):
    data = args.params if hasattr(args, "params") else args
    raw = _find_text(data)
    if not raw:
        return {"error": f"没找到天气文本，实际收到：{data}"}

    city = raw.split("：", 1)[0].strip()
    weather = raw.split("：", 1)[1].split("。", 1)[0].strip()
    temp_match = re.search(r"温度\s*(-?\d+\.?\d*)\s*°C", raw)
    temp = temp_match.group(1) if temp_match else ""

    rain_keywords = ("雨", "雪", "雷", "阵雨", "暴雨")
    is_rain = any(k in weather for k in rain_keywords)

    return {
        "city": city,
        "weather": weather,
        "temp": temp,
        "is_rain": is_rain,
    }
```

## 4\.5　接不通 / 不听话？快速排查

遇到问题别慌，对照下面几条基本都能解决：

- **右上角选不到模型：**回第 3 课确认模型已接入并设了默认推理模型。

- **回答不按人设来：**把提示词写得更具体、更有约束（第 5 课），别只写一句话。

- **别人访问还是旧版：**你没点“发布 / 更新”，或访问的是旧链接。

- **回答太啰嗦 / 太短：**在提示词里明确字数，如“控制在 150 字以内”。

|**❓ 常见疑问**<br>问：聊天助手会一直记得我之前说的话吗？<br>答：在同一段对话里会（系统会把历史一起发给模型）；但关掉重开一段新对话，它就不记得上一段了。如果要长期记住特定信息，需要用知识库或更复杂的编排来实现。|
|---|

|**❓ 常见疑问**<br>问：聊天助手和“对话流”有什么区别？<br>答：聊天助手（基础编排）是“一个提示词 \+ 一个模型”的简单版；对话流是它的加强版，能在每次回答前插入查资料、判断分支等多个步骤。先用简单的，不够了再升级。|
|---|

|**✏️ 课堂练习**<br>1\. 跟着 4\.3 节做出“英语学习小助手”，并和它连续对话 5 轮，感受“多轮记忆”。<br>2\. 只改提示词，把它改造成“健身教练”或“菜谱助手”，观察人设变化带来的回答差异。<br>3\. 给它加一句开场白和 3 个开场问题，调一次温度，再正确发布一次。<br>4\. 故意把温度调到最高和最低，各问同一个问题，对比回答风格的差别。|
|---|
